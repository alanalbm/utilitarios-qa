Smoke, sanity e regressão: diferenças práticas

Escolha a suíte certa para cada momento do ciclo de entrega.

Atualizado em 2026-07-218–12 min

Por que isso importa

Escolha a suíte certa para cada momento do ciclo de entrega. Este guia organiza o assunto em decisões práticas para planejamento, execução e comunicação de testes.

Uma abordagem prática

  1. Use smoke para verificar se a versão é testável.
  2. Use sanity para validar uma área após mudança limitada.
  3. Use regressão para procurar impacto em funcionalidades existentes.
  4. Organize suítes por risco e duração.
  5. Revise o conjunto para remover duplicidade e testes sem sinal.
Exemplo de aplicação: Escolha uma funcionalidade fictícia, aplique os passos abaixo e registre o motivo de cada cenário. O objetivo não é aumentar a quantidade de testes, mas melhorar a informação que eles entregam.

Erros comuns

  • Começar pela ferramenta antes de entender o risco.
  • Criar muitos casos semelhantes sem objetivo claro.
  • Ignorar dados, ambiente e condições que afetam o resultado.
  • Automatizar sem uma mensagem de falha útil.

Conclusão

Smoke, sanity e regressão: diferenças práticas funciona melhor quando a técnica é adaptada ao contexto, revisada com o time e ligada a riscos reais do produto.
