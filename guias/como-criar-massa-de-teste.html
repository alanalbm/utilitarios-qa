Como criar massas de teste seguras e reutilizáveis

Aprenda a separar dados fictícios, dados sintéticos e informações reais em testes de software.

Uma massa de teste é o conjunto de dados usado para executar um cenário. Ela precisa representar o comportamento que será validado, mas não deve expor pessoas reais nem criar dependência desnecessária de um ambiente.

1. Comece pela regra de negócio

Antes de criar dados, liste as condições importantes: valor mínimo e máximo, campos obrigatórios, estados permitidos, duplicidade, datas e permissões. Uma massa é útil quando existe para demonstrar uma regra específica.

2. Prefira dados sintéticos

Dados sintéticos são criados especialmente para testes. Use domínios reservados como example.test, telefones claramente fictícios e identificadores gerados. Evite copiar planilhas de clientes ou bancos de produção.

3. Torne a massa reproduzível

Registre como os dados foram criados, quais pré-condições existem e o que precisa ser limpo depois. Em automação, prefira factories, builders ou scripts versionados em vez de valores espalhados nos steps.

4. Separe cenários positivos e negativos

  • Dados válidos e completos.
  • Campos ausentes ou vazios.
  • Limites de tamanho e valor.
  • Formato correto com regra de negócio inválida.
  • Registros duplicados ou já processados.
Boa prática: a massa deve ajudar a entender o teste. Nomes como “clienteAtivoSemDebito” comunicam melhor do que sequências de números sem contexto.
