Glossário de QA e testes de software

60 termos de QA, automação, APIs, acessibilidade e desenvolvimento explicados de forma direta.

API

Interface que permite a comunicação entre sistemas por regras e formatos definidos.

Assert

Verificação que compara o resultado observado com uma expectativa.

Automação de testes

Uso de código e ferramentas para executar verificações repetíveis.

BDD

Desenvolvimento orientado a comportamento, com exemplos compartilhados entre negócio e tecnologia.

Boundary value analysis

Técnica que concentra casos nos limites e em valores imediatamente próximos.

Bug

Comportamento do produto diferente do esperado ou necessário.

Caso de teste

Conjunto de objetivo, condições, dados, passos e resultado esperado.

CI/CD

Práticas de integração e entrega contínuas com feedback automatizado.

Cobertura

Indicação do que foi exercitado por testes; precisa ser interpretada com risco e qualidade.

Contrato de API

Definição de operações, campos, tipos, respostas e compatibilidade de uma API.

Critério de aceite

Condição que ajuda a determinar se uma necessidade foi atendida.

Dados de teste

Informações preparadas para reproduzir condições e regras sem depender de produção.

Defeito

Problema identificado no produto, requisito, código ou configuração.

Definition of Done

Acordo sobre condições mínimas para considerar um item concluído.

Dublê de teste

Substituto controlado de uma dependência, como mock, stub ou fake.

E2E

Teste ponta a ponta que percorre um fluxo integrado semelhante ao uso real.

Equivalence partitioning

Técnica que agrupa entradas com comportamento esperado semelhante.

Evidência

Registro útil do contexto, ação e resultado de uma execução.

Exploratory testing

Aprendizado, desenho e execução de testes acontecendo de forma integrada.

Fake

Implementação simplificada, porém funcional, usada no lugar de uma dependência real.

Feature flag

Mecanismo para ativar ou desativar comportamento sem novo deploy completo.

Flaky test

Teste que alterna entre sucesso e falha sem mudança relevante no produto.

Gherkin

Linguagem estruturada com palavras como Dado, Quando e Então para descrever comportamento.

Idempotência

Propriedade em que repetir a mesma operação produz efeito equivalente ao primeiro processamento.

Integração

Interação entre componentes, serviços, bancos ou sistemas externos.

JSON Schema

Vocabulário para descrever e validar estrutura e regras de documentos JSON.

Mock

Dublê configurado para verificar interações esperadas.

Mutation testing

Técnica que altera o código de propósito para verificar se os testes detectam a mudança.

Observabilidade

Capacidade de compreender o estado interno por logs, métricas e traces.

OpenAPI

Especificação para descrever APIs HTTP de forma legível por pessoas e ferramentas.

Pairwise

Técnica combinatória que busca cobrir pares de valores com menos combinações.

Performance testing

Avaliação de tempo, capacidade, estabilidade e recursos sob carga.

Pirâmide de testes

Modelo que sugere mais verificações rápidas em níveis baixos e menos testes lentos de interface.

Plano de testes

Documento ou acordo que organiza objetivos, escopo, riscos, estratégia e critérios.

Pré-condição

Estado que precisa existir antes da execução de um cenário.

QA

Atividades que ajudam a prevenir problemas e melhorar a qualidade do produto e do processo.

Regressão

Verificação de que mudanças não prejudicaram comportamentos existentes.

Requirement traceability

Relação entre requisitos, riscos, testes, execuções e defeitos.

REST

Estilo arquitetural frequentemente usado por APIs HTTP.

Risk-based testing

Priorização de testes conforme impacto e probabilidade de falha.

Sanity test

Verificação focada de uma área após mudança limitada.

Selector CSS

Expressão que localiza elementos HTML pela estrutura, atributos ou classes.

Severidade

Grau do impacto causado por um defeito.

Shift left

Antecipação de atividades de qualidade e feedback no ciclo de desenvolvimento.

Shift right

Aprendizado e validação em operação com monitoramento e experimentos controlados.

Smoke test

Conjunto rápido que verifica se uma versão está estável o suficiente para continuar os testes.

Sprint

Ciclo curto de trabalho usado em abordagens ágeis.

State transition testing

Técnica que valida estados, eventos e transições permitidas ou proibidas.

Stub

Dublê que fornece respostas controladas para uma dependência.

Test charter

Missão curta que orienta uma sessão de teste exploratório.

Test data masking

Transformação de dados para reduzir exposição de informações sensíveis.

Test strategy

Conjunto de decisões sobre como qualidade e riscos serão abordados.

Testability

Facilidade de controlar, observar e diagnosticar o comportamento de um sistema.

Trace

Registro distribuído que acompanha uma operação entre serviços.

Unit test

Teste focado em uma pequena unidade de comportamento com dependências controladas.

Usabilidade

Facilidade com que pessoas alcançam objetivos com eficiência e compreensão.

Valor limite

Ponto mínimo, máximo ou de transição em uma regra.

Visual regression

Comparação visual para detectar mudanças inesperadas na interface.

WCAG

Diretrizes internacionais de acessibilidade para conteúdo Web.

XPath

Linguagem para localizar nós em documentos XML e elementos em estruturas HTML.
